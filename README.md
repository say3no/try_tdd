# try_tdd

## 読書メモ

### まえがき

>*レッド・グリーン・リファクタリング。それがTDDのマントラだ。*

テスト駆動開発のルールはたったのふたつ。
 - テストが失敗した時、新しいコードを書く。
 - 重複とを取り除く。

何を作りたいかをテストという形で明確化(レッド）させ、実装し(グリーン）、改善(リファクタリング）する。
